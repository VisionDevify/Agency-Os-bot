# Team Rollout

Sprint 16 makes Fortuna OS easier for real team members to use every day.

## Rollout Principles

- Show each role only the pages they need.
- Keep owner/admin controls available without exposing internals to staff.
- Make every screen readable by non-technical users.
- Bundle low-priority notifications so daily work does not become noisy.
- Keep critical alerts immediate.
- Track onboarding readiness so managers can see who is ready for live work.

## Role Homes

- Owner/Admin: Setup Agency, First Day Plan, Executive Command Center, Intelligence, Models, Accounts, Proxies, Operations, Reports, Automation, Settings.
- Manager/Chatter Manager: Team, Models, Tasks, Incidents, Opportunities, Reports, Manager QA, Help.
- Chatter/Senior Chatter: My Models, My Opportunities, My Tasks, Availability, Help.
- VA: My Models, My Accounts, My Tasks, Availability, Help.
- Model/Client: My Dashboard, My Accounts, My Reports, My Team.

## Team QA

`team_onboarding_checklists` tracks rollout readiness:

- role assigned
- timezone confirmed
- availability configured
- help center viewed
- onboarded

The readiness score is for operational support, not punishment.

## Activation Checklist

- Invite team member to the bot.
- Let them complete localization onboarding.
- Assign role.
- Confirm timezone.
- Confirm availability.
- Ask them to open Help Center.
- Mark onboarded when ready.

## Sprint 19 Setup Rollout

The first live rollout should start with Owner Home -> Setup Agency.

Recommended order:

1. Create the first model/brand.
2. Add platform account records.
3. Assign manager/chatter/VA team members.
4. Add creator watch starters.
5. Create starter opportunities.
6. Review Manager QA for missing assignments, pending users, timezone gaps, and users without work.

Demo Seed Mode is owner-only. It is useful for training screens, but demo records should be cleared before real team operations.
